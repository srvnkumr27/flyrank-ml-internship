01

Abstract

Organizations managing large websites often struggle to determine which content should be refreshed first to maintain search performance. This study proposes a machine learning–based content refresh opportunity scoring framework using the FlyRank ML Internship dataset. Historical search performance, website engagement, and traffic source metrics were transformed into engineered features and evaluated using supervised machine learning models with time-aware validation to avoid data leakage. The proposed framework ranks content based on its predicted refresh priority and provides interpretable reason codes for each recommendation. The resulting decision-support system enables data-driven prioritization of content updates while maintaining reproducible and privacy-safe research practices.

02

Introduction

Search visibility is one of the primary drivers of organic website traffic. As websites grow to hundreds or thousands of pages, manually reviewing every page becomes impractical. Pages that once performed well may gradually lose visibility because of outdated information, changing search intent, or increased competition.

The objective of this research is to develop a repeatable machine learning framework that identifies content requiring review before significant performance loss occurs. Instead of attempting to explain search engine algorithms or establish causal relationships, this work provides observational evidence from historical search and engagement data to support content refresh decisions.

Research question:

Can historical search performance and engagement signals be used to identify and rank webpages that should be prioritized for content refresh?

04

Methodology

Research design

The project follows a supervised machine learning workflow for content refresh opportunity scoring. The overall workflow consists of:

  1. Data extraction using DuckDB over the FlyRank warehouse
  2. Data cleaning and preprocessing
  3. Feature engineering
  4. Target label creation
  5. Model training
  6. Model evaluation
  7. Ranked recommendation generation

Feature engineering

Several derived features were created from historical search performance. These engineered variables capture both historical performance and temporal behavior rather than relying solely on raw metrics. Examples include:

  • Click-through Rate (CTR)
  • Engagement Rate
  • Average Engagement Time
  • Organic Traffic Share
  • AI Traffic Share
  • Click Trend
  • Impression Trend
  • Position Trend
  • Session Growth Rate
  • Rolling Moving Averages
  • Performance Volatility

Target definition

A refresh opportunity label was created using future performance windows. Content experiencing sustained declines in search performance over the evaluation window was labeled as requiring review. The target definition was designed using future observations to reduce information leakage between training and evaluation periods.

Baseline model

The baseline model used Logistic Regression because of its interpretability and strong performance on structured tabular data.

Candidate models

  • Random Forest
  • LightGBM

Validation strategy

A time-aware train/validation/test split was used to preserve temporal order and reduce data leakage. Model performance was evaluated on future observations rather than randomly shuffled samples. Evaluation metrics included:

  • Precision
  • Recall
  • F1 Score
  • ROC-AUC (where applicable)

Results

The machine learning models were evaluated using identical validation datasets. The baseline Logistic Regression model established a reference point for comparison. Tree-based ensemble methods demonstrated improved predictive performance by capturing nonlinear relationships among search and engagement signals.

Feature importance analysis indicated that historical click trends, impression trends, engagement metrics, and average search position contributed most strongly to refresh opportunity predictions.

The proposed scoring framework successfully ranked content according to predicted refresh priority, allowing pages to be grouped into actionable review categories.

Limitations

Read this as observed / directional, not causal

This research has several limitations, listed below.

  • The findings are observational and should not be interpreted as causal relationships.
  • Search engine ranking systems are influenced by many external factors that are not captured within the available dataset.
  • Labels are generated using predefined business rules rather than manual editorial review.
  • Model performance depends on the available historical data and may vary across websites and industries.
  • Recommendations should support human decision-making rather than replace editorial judgment.

Ranked Recommendations

The proposed framework generates a prioritized action list for content review. Each recommendation is accompanied by evidence-based reason codes generated from the underlying search and engagement signals.

PriorityRecommendationReason Code
High Refresh content immediately Declining clicks, declining impressions, worsening average position
High Update titles and metadata High impressions with low click-through rate
Medium Improve internal linking Stable visibility with decreasing engagement
Medium Expand or revise content Organic sessions decreasing while impressions remain stable
Low Continue monitoring Stable search performance and engagement
